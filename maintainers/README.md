# Maintainers

Maintainers are stewards of the club's work.

## Maintainer Types

| Type | Main job |
|---|---|
| Club Maintainer | Owns shared rules, templates, archive policy, and license policy |
| Project Maintainer | Owns one project repo, reviews PRs, triages issues, and approves merges |
| Area Maintainer | Looks after one area such as frontend, backend, docs, QA, or release work |

## Rules

- No self-merges
- At least one reviewer for every pull request
- Weekly health checks for maintainer activity
- If a maintainer misses two weekly check-ins, pause the role and replace it
- Every project should have a primary maintainer and a backup maintainer
- Keep handoff notes when someone leaves the club or changes roles

## Good Maintainer Behavior

- Be clear
- Be fair
- Be available
- Leave comments that help the next person move forward
- Keep the project welcoming to beginners
