# Week 1 - Planning and Team Formation

Week 1 is about starting clean and choosing well.

## Goal

- Form the teams
- Choose the leaders
- Pick a useful project
- Create the repository
- Build the first roadmap

## What Leaders Do

- Introduce themselves and their style
- Keep the meeting on time
- Help the team stay focused on a useful project
- Make sure everyone gets a role and a first task

## What Members Do

- Vote for the leaders anonymously
- Choose a role title
- Help decide the project idea
- Start the issue board

## What Beginners Do

- Join onboarding
- Learn the repo structure
- Take a starter task
- Ask for help early

## Deliverables

- Team list
- Role list
- Public repository
- Roadmap
- Task board

## Done When

- Everyone knows their role
- The project choice is written down
- The team has at least one starter issue per beginner
- The repository is ready for Week 2 work

## Important Rules

- Use an anonymous first-day election for leaders
- After that, use weekly confidence reviews instead of random leadership changes
- Keep the project scope small enough to finish in one month
- Start with a useful public problem
