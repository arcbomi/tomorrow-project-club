# Week 3 - Expansion and Stabilization

Week 3 is where the project becomes stronger, cleaner, and easier to present.

## Goal

- Improve the project
- Fix issues
- Polish the user experience
- Prepare the presentation

## What Leaders Do

- Check that the team is still on schedule
- Decide what gets polished first
- Keep the work focused on the MVP plus the best useful extras
- Make sure docs are being written

## What Members Do

- Add features carefully
- Fix bugs
- Improve the UI and usability
- Test the project more seriously

## What Beginners Do

- Help with UI fixes
- Improve docs
- Run tests
- Translate or polish text where needed

## Deliverables

- Stable demo
- Better UI
- Documentation draft
- Release candidate plan

## Done When

- The project feels stable enough to show to outsiders
- The docs explain how to use the project
- The team knows what is still missing before Week 4

## Important Rules

- Do not let Week 3 become a brand-new project
- Keep polish and testing as important as new features
- Write down any risky changes before making them
