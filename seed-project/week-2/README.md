# Week 2 - Core Development

Week 2 is where the team proves the project can actually work.

## Goal

- Build the MVP
- Get the first real demo working
- Find blockers early

## What Leaders Do

- Keep the scope small
- Remove blockers fast
- Make sure every person has a useful task

## What Members Do

- Start implementation
- Merge the first useful changes
- Report blockers honestly

## What Beginners Do

- Work on a starter issue
- Pair with a mentor
- Learn how issues and pull requests move through the repo

## Deliverables

- Working prototype
- Basic UI
- First internal demo
- Blocker list

## Done When

- The project can be shown to another person without a long explanation
- The team has at least one merged PR
- The MVP scope is written down
- The weekly demo includes real progress, not just promises
