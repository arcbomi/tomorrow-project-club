# Beginner Pipeline

New members should always have a clear way to get started.

## Stage 1: Join a Team and Orientation

- Always join a team first
- After you join, ask the team leader for a job title that fits the work
- Meet the team
- Learn the project goal
- Learn GitHub basics
- Learn how issues and pull requests work
- Learn the club privacy rules

## Stage 2: Starter Task

- Give the beginner a small issue right away
- Good starter tasks are docs, QA, UI polish, translations, or a tiny fix
- The task should be small enough to finish in one session

## Stage 3: Paired Pull Request

- Open the first PR with help from a mentor or maintainer
- Review the comments together
- Learn how to revise work without stress

## Stage 4: Independent Task

- After one merged PR, give the beginner a slightly larger issue
- Keep support available, but let the beginner work more independently

## Stage 5: Contributor Growth

- After two successful PRs and one demo contribution, the member can take on a larger role
- Growth should come from steady progress, not pressure

## Mentor Duties

- Explain the project in plain language
- Help the beginner pick the right issue
- Check that the beginner does not get stuck for too long
- Celebrate the first finished task
