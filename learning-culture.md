# Learning Culture

Project Club is not only about building a project.

It is also about becoming stronger engineers.

---

## Learning Is Part of the Work

We do not expect everyone to know every technology.

It is normal to:

* Ask questions.
* Make mistakes.
* Learn new concepts.

What is not acceptable is staying silent and avoiding problems.

---

## Teach, Don't Just Solve

When helping a teammate:

* Explain the approach.
* Share your reasoning.
* Point them in the right direction.
* Encourage independent problem-solving.

The goal is not to finish someone's task.

The goal is to help them solve similar problems in the future.

---

## Mistakes Are Part of the Process

Bugs, incorrect assumptions, and poor decisions are inevitable.

Every mistake is an opportunity to improve:

* Knowledge
* Documentation
* Processes
* Team understanding

We analyze mistakes without blaming people.

---

## Everyone Should Understand the Project

Every team member should understand:

* The problem the project solves.
* The MVP goals.
* The high-level architecture.
* The purpose of each major component.

Even if they did not implement a particular module.

---

## Document Knowledge

If you learn something important:

* Document it.
* Share it with the team.
* Update project documentation when appropriate.

Knowledge should belong to the team, not to individuals.

---

## Growth Matters More Than Code Volume

Contribution is not measured only by lines of code.

Valuable contributions include:

* Writing code
* Reviewing code
* Improving documentation
* Finding bugs
* Asking useful questions
* Helping teammates

The goal is project success, not personal statistics.

---

## The 1% Rule

Every team member should become slightly stronger every week.

If, after four weeks, everyone knows more than they did at the beginning, the team has succeeded regardless of the competition result.

---

## Leave Things Better Than You Found Them

Whenever you touch part of the project:

* Improve clarity.
* Improve documentation.
* Improve code quality.
* Improve understanding.

Small improvements accumulate into significant progress.
