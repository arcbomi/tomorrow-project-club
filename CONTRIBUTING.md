# Contributing

Thanks for helping improve the club handbook.

## Before You Change Anything

- Read [guide.md](guide.md) first
- Check the matching folder, such as `seed-project/`, `infra/`, or `maintainers/`
- Keep the change small and easy to review
- Use plain English
- Do not add private student or school data

## What Good Contributions Look Like

- One clear issue or improvement per pull request
- Short, readable wording
- Updated links when a file moves
- Consistent terms across files
- Changes that help beginners understand the club faster

## Suggested Workflow

1. Open an issue or write down the change idea.
2. Update the right markdown file.
3. Check whether related docs need the same rule.
4. Ask for review.
5. Merge only after the content is clear and consistent.

## Style Rules

- Use lowercase kebab-case for folder names
- Keep headings short
- Prefer tables or checklists when they make the rules easier to scan
- Explain special terms the first time they appear
- Keep the tone kind, practical, and beginner-friendly
